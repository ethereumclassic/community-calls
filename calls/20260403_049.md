### Next Fork EIPs

- [ECIP-1121](https://ecips.ethereumclassic.org/ECIPs/ecip-1121) still uses "Olympia" name and includes EIP-7935 despite consensus to change both — should we draft a fresh ECIP instead?
- EIP-7702: new tx type uses EIP-1559 fee fields — has this been tested on ETC's non-1559 chain?
- What is a reasonable mainnet target?

| EIP | 1121 | Notes |
|-----|:----:|-------|
| **[7823](https://eips.ethereum.org/EIPS/eip-7823) Upper bounds for MODEXP inputs** | 🔴 | **Should we add?** <br> Caps MODEXP inputs at 1024 bytes. Prevents abuse of underspecified upper bounds. Simple, low-risk hardening that complements 7883. |
| **[7939](https://eips.ethereum.org/EIPS/eip-7939) CLZ (Count Leading Zeros) opcode** | 🔴 | **Should we add?** <br> New opcode for counting leading zeros. Useful for bit manipulation in contracts. Low complexity, no dependencies. |
| **[7935](https://eips.ethereum.org/EIPS/eip-7935) Default gas limit to 60M** | 🟢 | **Should we remove?** <br> Consensus in call 48 to defer; PR still not submitted. ETC doesn't need increased block space. Raises risk surface without clear benefit. |
| **[2935](https://eips.ethereum.org/EIPS/eip-2935) Historical block hashes in state** | 🟢 | **Should we remove?** <br> Writes parent hashes into a system contract every block. Built for stateless clients and beacon-chain bridges — neither exists on ETC's PoW architecture. Introduces system contract deployment pattern not used elsewhere in ETC. |
| **[7623](https://eips.ethereum.org/EIPS/eip-7623) Floor data gas / calldata cost increase** | 🟢 | **Should we remove?** <br> Sets minimum gas cost per tx based on calldata size. Designed to push L2s toward blobs — ETC has no blob support and no L2 rollup ecosystem. DoS hardening effect is modest given ETC's lower block gas limit. |
| **[7825](https://eips.ethereum.org/EIPS/eip-7825) Transaction gas limit cap at 2^24** | 🟢 | **Should we remove?** <br> Caps single tx at ~16.7M gas to bound worst-case ZK proof time. ETC's block gas limit (~8M) is already well below this cap, making it redundant. Worth revisiting if gas limit is raised. |
| **[7934](https://eips.ethereum.org/EIPS/eip-7934) RLP block size limit at 10 MiB** | 🟢 | **Should we remove?** <br> Hard cap on RLP-encoded block size. Includes 2 MiB margin for beacon chain wrapping that doesn't apply to ETC. Would need re-parameterization if kept. Blocks can't approach this size at current gas limits. |
| **[7702](https://eips.ethereum.org/EIPS/eip-7702) Set EOA Account Code** | 🟢 | **Should we remove?** <br> Introduces type 0x04 transactions that use EIP-1559 fee fields. ETC doesn't support type 0x02 (1559). Highest complexity EIP in the set — needs thorough testing on ETC's fee model before Mordor deployment. |
| [1153](https://eips.ethereum.org/EIPS/eip-1153) Transient storage (TSTORE/TLOAD) | 🟢 | Cancun; implemented in core-geth |
| [5656](https://eips.ethereum.org/EIPS/eip-5656) MCOPY instruction | 🟢 | Cancun; implemented in core-geth |
| [6780](https://eips.ethereum.org/EIPS/eip-6780) SELFDESTRUCT restricted to same tx | 🟢 | Cancun; implemented in core-geth |
| [2537](https://eips.ethereum.org/EIPS/eip-2537) BLS12-381 precompiles | 🟢 | Prague; implemented in core-geth |
| [7883](https://eips.ethereum.org/EIPS/eip-7883) MODEXP gas cost increase | 🟢 | Osaka; corrects underpriced math ops |
| [7951](https://eips.ethereum.org/EIPS/eip-7951) secp256r1 (P-256) signature verification precompile | 🟢 | Osaka; enables passkey-based wallets |
| [7910](https://eips.ethereum.org/EIPS/eip-7910) eth_config RPC method | 🟢 | Returns active fork rules; discussed in calls 46–48 |
| [7642](https://eips.ethereum.org/EIPS/eip-7642) Remove bloom filters from receipts | 🔴 | Dropped in call 47 via PR #563 |

### Bitmain AMA Debrief

Only the first half of the AMA was recorded due to a technical issue. The full recording will be uploaded soon. The [first section is available here](https://x.com/ETCGrantsDao/status/2036001252123811929).

During the AMA, [nolympia.dev](https://nolympia.dev) was introduced — a petition opposing the Olympia treasury proposal (ECIPs 1111–1119), which would redirect transaction fees to an on-chain funding contract. The petition argues this contradicts ETC's foundational principles by introducing a protocol-enforced tax on miners and a complex governance system. It currently has 20 signatories.
